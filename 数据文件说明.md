## 根目录下 `checkpoints` 与 `json` 文件说明

> 说明：当前仓库中只包含 `json` 目录，`checkpoints` 目录需要你从 README 中的 OneDrive 链接下载并解压到项目根目录。下面分别说明它们的作用和典型使用场景。

---

### 一、`checkpoints` 目录（模型权重）

路径：`E:\work\adv_attack\checkpoints\`

这些文件是训练好的模型权重（`.pt`），在生成对抗样本和评估时被加载。

- **`coco_cos.pt`**
  - **作用**：COCO 数据集、以 Cosine 损失训练的解码器权重。
  - **常用场景**：
    - 单张 demo：`demo.py --decoder_path checkpoints/coco_cos.pt`
    - 批量生成 COCO 相关对抗图像：`generate_adv_img.py --decoder_path checkpoints/coco_cos.pt --dataset coco_caption` 等。

- **`coco_bi.pt`**
  - **作用**：COCO 检索任务、以 Bi-Contrastive 损失训练的解码器权重。
  - **常用场景**：在 `scripts/generate_adv_img.sh` 中用于 `coco_retrieval` 任务的对抗样本生成。

- **`flickr30k_bi.pt`**
  - **作用**：Flickr30K 检索任务的解码器权重。
  - **常用场景**：在 `scripts/generate_adv_img.sh` 中用于 `flickr30k` 任务的对抗样本生成。

- **`snli_ve_cos.pt`**
  - **作用**：SNLI-VE 多模态分类任务（视觉蕴含）的解码器权重。
  - **常用场景**：在 `scripts/generate_adv_img.sh` 中用于 `snli_ve` 任务的对抗样本生成。

- **`pre-trained.pt`**
  - **作用**：在 `finetune_ddp.py` 中用作微调的初始预训练权重。
  - **常用场景**：运行 `scripts/finetune_ddp.sh` 进行下游任务微调时加载。

> 注：OneDrive 压缩包中可能还包含其他 `.pt` 文件（对应更多模型或训练阶段），用法与上述类似：在运行脚本时通过 `--decoder_path` 或配置文件中的路径进行指定。

---

### 二、`json` 目录（标注与评估配置）

路径：`E:\work\adv_attack\json\`

这些 JSON 文件主要有两类：

1. **`*_target.json`**：攻击目标/原始标注，用于生成对抗样本时提供文本或图像标注。
2. **`*_adv.json`** / `*_gt_adv.json`：对抗样本对应的标注或评估用 ground truth，在下游评估脚本中使用。

逐个说明如下。

---

#### 1. COCO 检索相关

- **`coco_retrieval_target.json`**
  - **作用**：COCO 图文检索任务的目标标注（原始 clean 场景），包含图像与文本配对信息。
  - **使用场景**：
    - 在大规模对抗样本生成阶段，作为目标文本/图像信息的来源。

- **`coco_retrieval_adv.json`**
  - **作用**：COCO 图文检索任务的对抗样本标注，描述对抗图像与文本的对应关系。
  - **使用场景**：
    - 评估对抗样本在图文检索任务上的效果：
      - UI 中 **“图文检索（retrieval）”** 页的 `json_path` 默认指向该文件。
      - 或手动运行 `retrieval.py --json_path json/coco_retrieval_adv.json`。

---

#### 2. COCO 图像描述（caption）相关

- **`coco_caption_target.json`**
  - **作用**：COCO caption 任务的目标标注（原始 clean 场景），包含每张图的目标描述。
  - **使用场景**：
    - 作为对抗样本生成时的目标描述输入，用于驱动解码器生成特定语义的噪声。

- **`coco_caption_adv.json`**
  - **作用**：对抗样本对应的 caption 标注，描述在对抗图像上希望得到/观察到的描述。
  - **使用场景**：
    - 后续对抗 caption 评估时，作为对抗场景的标注文件输入。

- **`coco_caption_test_gt_adv.json`**
  - **作用**：COCO caption 任务的 **对抗评估 ground truth**，用于和模型生成的描述一起计算 BLEU、CIDEr 等指标。
  - **使用场景**：
    - 评估图像描述任务：
      - UI 中 **“图像描述（caption）”** 页的 `gt_path` 默认指向该文件。
      - 或手动运行：
        - `caption.py --json_path json/coco_caption_adv.json --gt_path json/coco_caption_test_gt_adv.json ...`

---

#### 3. Flickr30K 检索相关

- **`flickr30k_target.json`**
  - **作用**：Flickr30K 图文检索任务的目标标注（原始 clean 场景）。
  - **使用场景**：
    - 生成针对 Flickr30K 的对抗检索样本时，作为目标文本/图像信息的来源。

- **`flickr30k_adv.json`**
  - **作用**：Flickr30K 对抗样本的标注文件。
  - **使用场景**：
    - 在使用 LAVIS 对 Flickr30K 对抗样本进行检索评估时提供标注信息（对应 `retrieval.py` + Flickr 配置）。

---

#### 4. SNLI-VE 多模态分类相关

- **`snli_ve_target.json`**
  - **作用**：SNLI-VE（视觉自然语言推理）任务的目标标注（clean 场景），包含图像与文本的蕴含关系标签（entailment / neutral / contradiction）。
  - **使用场景**：
    - 生成针对 SNLI-VE 的对抗样本时，提供原始图像与文本标签信息。

- **`snli_ve_adv.json`**
  - **作用**：SNLI-VE 对抗样本的标注文件。
  - **使用场景**：
    - 多模态分类对抗评估：
      - UI 中 **“多模态分类（classification）”** 页的 `json_path` 默认指向该文件。
      - 或手动运行：
        - `classification.py --json_path json/snli_ve_adv.json ...`

---

### 三、典型工作流程中这些文件的使用顺序

1. **生成对抗样本**
   - 选择对应任务的 decoder 权重（例如 `checkpoints/coco_cos.pt`）。
   - 使用 `*_target.json` 提供目标标注（例如 `coco_caption_target.json`）。
   - 在 UI 的 **“生成对抗样本”** 页面，或用 `generate_adv_img.py` 批量生成对抗图像并保存到 `outputs/...`。

2. **构建对抗标注文件**
   - 生成对抗图像的脚本会同时写出相应的 `*_adv.json`（例如 `coco_retrieval_adv.json`、`snli_ve_adv.json`）。

3. **评估对抗样本效果**
   - 图文检索：使用 `coco_retrieval_adv.json` / `flickr30k_adv.json`，配合 LAVIS 配置和对抗图像目录，运行 `retrieval.py`。
   - 多模态分类：使用 `snli_ve_adv.json`，运行 `classification.py`。
   - 图像描述：使用 `coco_caption_adv.json` 作为对抗标注，`coco_caption_test_gt_adv.json` 作为 ground truth，运行 `caption.py`。
